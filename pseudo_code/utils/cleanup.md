# cleanup.c 疑似コード

## 関数: cleanup_data
- **概要**:
  - シェルのデータ構造体と関連リソースを解放する。
- **引数**:
  - `data`: シェルの主要データ構造体
- **処理**:
  1. `data` が `NULL` なら何もしない。
  2. `data->cmds` があれば `free_cmds` で解放する。
  3. `data->env` があれば `free_env` で解放する。
  4. `stdin_backup` が有効 (`!= -1`) なら閉じる。
  5. `stdout_backup` が有効 (`!= -1`) なら閉じる。

## 関数: free_redirs
- **概要**:
  - リダイレクト構造体のリストを解放する。
- **引数**:
  - `redirs`: 解放するリダイレクトリスト
- **処理**:
  1. リストを走査するループ:
     - 現在のノードを `tmp` に保存する。
     - 次のノードへ進む。
     - ヒアドキュメントの場合 (`TOKEN_REDIR_HEREDOC`):
       - ファイル名が `/tmp/.heredoc_` で始まる場合、`unlink` でファイルを削除する。
     - ファイル名 (`file`) を解放する。
     - ノード自体 (`tmp`) を解放する。

## 関数: free_cmds
- **概要**:
  - コマンド構造体のリストを解放する。
- **引数**:
  - `cmds`: 解放するコマンドリスト
- **処理**:
  1. リストを走査するループ:
     - 現在のノードを `tmp` に保存する。
     - 次のノードへ進む。
     - 引数配列 (`args`) を `ft_free_split` で解放する。
     - クォートタイプ配列 (`quote_types`) を解放する。
     - リダイレクトリスト (`redirs`) を `free_redirs` で解放する。
     - ノード自体 (`tmp`) を解放する。

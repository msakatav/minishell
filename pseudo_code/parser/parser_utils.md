# parser_utils.c 疑似コード

## 関数: new_cmd
- **概要**:
  - 新しいコマンド構造体を作成し、初期化する。
- **引数**: なし
- **処理**:
  1. `t_cmd` 構造体のメモリを確保する。
  2. メンバ (`args`, `quote_types`, `redirs`, `next`) を `NULL` に初期化する。
  3. 作成した構造体を返す。

## 関数: add_cmd
- **概要**:
  - コマンドリストの末尾に新しいコマンドを追加する。
- **引数**:
  - `cmds`: コマンドリストへのポインタ
  - `new`: 追加する新しいコマンド
- **処理**:
  1. `new` が `NULL` なら何もしない。
  2. リストが空なら、先頭に設定する。
  3. リストの末尾まで移動し、最後に連結する。

## 関数: new_redir
- **概要**:
  - 新しいリダイレクト構造体を作成し、初期化する。
- **引数**:
  - `type`: リダイレクトの種類
  - `file`: ファイル名
- **処理**:
  1. `t_redir` 構造体のメモリを確保する。
  2. `type` と `file` を設定する。
  3. `next` を `NULL` に初期化する。
  4. 作成した構造体を返す。

## 関数: add_redir
- **概要**:
  - リダイレクトリストの末尾に新しいリダイレクトを追加する。
- **引数**:
  - `redirs`: リダイレクトリストへのポインタ
  - `new`: 追加する新しいリダイレクト
- **処理**:
  1. `new` が `NULL` なら何もしない。
  2. リストが空なら、先頭に設定する。
  3. リストの末尾まで移動し、最後に連結する。

## 関数: free_cmds
- **概要**:
  - コマンドリストとそのメンバ（引数、リダイレクトなど）のメモリをすべて解放する。
- **引数**:
  - `cmds`: 解放するコマンドリスト
- **処理**:
  1. コマンドリストを走査するループ:
     - 現在のコマンドを `tmp_cmd` に保存する。
     - 次のコマンドへ進む。
     - `args` (文字列配列) を解放する。
     - `quote_types` (整数配列) を解放する。
     - リダイレクトリスト `redirs` を走査して解放する:
       - ファイル名を解放する。
       - ノード自体を解放する。
     - コマンド構造体自体を解放する。
